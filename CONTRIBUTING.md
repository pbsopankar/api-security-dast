# Contributing to API Security DAST

Thank you for your interest in contributing to the API Security DAST project! To ensure a smooth collaboration, please follow the guidelines outlined below.

## Contribution Guidelines
- **Fork the repository.**
- **Create a new branch.** For example, `feature/my-new-feature`.
- **Make your changes** and ensure they are well-documented and tested.
- **Submit your pull request**, and we will review it as soon as possible.

## Development Setup
1. **Clone the repository:**
   ```bash
   git clone https://github.com/pbsopankar/api-security-dast.git
   cd api-security-dast
   ```
2. **Install dependencies:**
   ```bash
   npm install  # If using Node.js
   # or
   pip install -r requirements.txt  # If using Python
   ```
3. **Set up environment variables** as needed for your development.

## Testing Requirements
- Ensure that all tests pass before submitting your changes. Run the tests using:
  ```bash
  npm test  # For Node.js
  # or
  pytest  # For Python
  ```
- Add new tests for any new feature or bug fix you introduce.

## Code Standards
- Follow the project's coding style guidelines.
- Write clear and concise commit messages.
- Ensure your code is linted before submission. You can use tools like ESLint, Pylint, or Prettier depending on the language used.

Thank you for your contributions! Together, we can make API Security DAST even better!
